CONCEPTOS BASICOS

Antes que nada hay que entender algunos conceptos

¿QUE ES UNA BASE DE DATOS?

Es un conjunto de datos que se almacenan en una base de datos para que se puedan manipular y utilizar.

¿QUE ES UN DATO?

Es una unidad singular de conocimiento, es la minima unidad de informacion, un dato puede ser: 6, otro dato puede ser: rojo, tambien puede ser: rueda, todo esos son datos sueltos que en si no nos dice nada por si solos, pero si decimos que "en un racimo de uvas habia 6 nada mas" ahi tenemos el dato 6 con mas datos y ahi la informacion tiene mas sentido, "el globo que me regalaron es de color rojo" y "las ruedas de la bici estaban desinfladas".

¿QUE ES UN REGISTRO?

Es una unidad de información que se almacena en una base de datos.

¿QUE ES UNA TABLA?

Es una colección de datos relacionados que se almacenan en una base de datos.

¿QUE ES UN CAMPO?

Es una columna de datos en una tabla.

TIPOS DE BASES DE DATOS

Tenemos bases de datos RELACIONALES (SQL) y bases de datos NO RELACIONALES (NoSQL)

  • SQL: MySQL, PostgreSQL, Oracle, SQLite
  • NoSQL: MongoDB, CouchDB, Redis, Cassandra

(no confundir una base de datos con por ejemplo MySQL o MongoDB, estas son solo gestores de bases de datos, uno carga los datos en estos dos gestores nada mas)

Hay que entender bien la diferencia entre las bases de datos, porque al empezar a armar una base de datos relacionales y cuando vamos por la mitad nos damos cuanta que era mejor hacerlo con una base de datos no relacionales nos vamos a querer cortar las venas con una galletita de agua, por eso hay que saber bien las diferencias y entenderlas

BASE DE DATOS RELACIONALES (SQL)

Se caracteriza por ser una coleccion ordenada de registros que se van a organizar en un conjunto de tablas.

Una TABLA es el elemento principal de una base de datos relacionales (es muy parecida a una hoja de calculos, los registros de informacion se van a formar en filas y columnas), las tablas se relacionan entre si y para ingresar a los datos, usamos el sistema de consulta estructurada (mejor conocido por sus siglas en ingles SQL Structured Query Language)

Con SQL vamos a poder obtener y consultar los datos de una forma mas organizada, siempre y cuando tengamos en cuenta cual es la estructura de la base de datos con la que estamos trabajando.

Para ello vamos a utilizar distintos comandos que SQL nos pone a dispocision para poder trabajar con la informacion que almacene nuestra base.

Estas tablas tambien se organizan atravez de identificadores (cada tabla dentro de la informacion que almacena va a elegir a un campo como identificador unico, este identificador unico es el que establece la relacion entre las tablas, tambien hace mas facil identificar la informacion dentro de la base de dato

Ejemplos de base de datos relacionales

  • MySQL
  • PostgreSQL
  • Oracle
